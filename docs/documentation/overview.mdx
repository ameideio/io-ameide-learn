# Documentation

Product documentation for using Ameide inside a customer organization.
The structure mirrors the **Customer perspective** in the Ameide portal,
so what you see here matches what you see when you log in.

## What's covered

- **Strategy** — Vision, portfolio, and value tracking.
- **Architecture** — Context, design, risk and compliance.
- **Delivery** — Programs, milestones, and outcomes.
- **Change** — Change management and adoption.
- **Support** — Service desk, ticketing, and customer-facing support.

Cross-cutting topics live under **Administration** (Identity & SSO,
roles, billing, audit, threads). New here? Start with
[Getting Started](/docs/getting-started/what-is-ameide).

## Who this is for

Executive sponsors, enterprise architects, project and program managers,
transformation leads, change managers, and support owners — anyone
running Ameide as a customer organization.

Building a partner practice on Ameide? See
[Partners](/docs/partners/overview). Looking for skill-building? See
[Training & Certifications](/docs/training/overview).
